🚀 The Optimization Journey

How we slashed execution time from 115 minutes down to just 35 minutes.

Initial Implementation

Execution Time: ~115 minutes (for 99 queries)

The original script ran synchronously. Each page load on Google Flights took ~70 seconds due to stealth operations and DOM waiting. We scraped routes one by one.

Failed Attempt: ThreadPoolExecutor

Result: Threading conflicts

We attempted to use a ThreadPoolExecutor to run 3 browser instances in parallel. However, Playwright's Sync API uses the main thread's asyncio loop under the hood, causing I/O operation on closed pipe errors when workers tried to close contexts concurrently.

Final Solution: Native Asyncio

Execution Time: ~35 minutes

We rewrote the Playwright scraper to use the playwright.async_api native to asyncio. By using asyncio.Semaphore(3), we capped concurrent headless browsers to 3. This fully leveraged the network I/O wait times and perfectly parallelized the workload without thread safety issues.
